[MEDIUM] ANSIStripper.swift:12-17 — Regex recompilation on every call
  csiRegex and oscRegex are computed properties (var, not lazy var), so
  NSRegularExpression is reconstructed on every invocation of strip().
  Regex compilation is expensive. Should be lazy stored properties.
  Example: `private static let csiRegex = try? NSRegularExpression(...)`

  internally (ConversationViewModel.appendChunk line 20). Harmless but
  redundant — one strip should be removed.

--- 3g. Functional Config Bug (noted but not REJECT-level) ---
[HIGH] HotkeyConfig.default uses modifiers: ["cmd", "shift"]
  but HotkeyManager.modifiersFromStrings() only handles:
    "command", "shift", "option", "control"
  The string "cmd" does NOT match any case, so the Command modifier is
  silently dropped from the default hotkey registration.
  Result: default hotkey fires on Shift+K instead of Cmd+Shift+K.
  Tests don't catch this because HotkeyManagerTests only verify init/toggle
  settability, not actual modifier registration.
  RECOMMENDATION: Fix Config.swift default to use "command" instead of "cmd",
  OR add "cmd" as an alias in modifiersFromStrings().
